# 游戏难度提升与知识点扩展方案

## 一、总体设计思路

### 当前问题
1. 谜题数量少，学生很快就能完成
2. 知识点考查不够深入
3. 缺乏失败惩罚，试错成本低
4. 没有多轮验证，深度思考不足

### 改进目标
1. **增加游戏时长**：从15分钟延长到30-45分钟
2. **深化知识考查**：每个方法论至少3-5个知识点
3. **提高决策成本**：错误选择有明显惩罚
4. **增加思考深度**：多轮验证、案例分析

---

## 二、失败惩罚机制

### 1. 分数惩罚系统
```
错误次数 | 扣分 | 质量评级影响
--------|------|-------------
第1次   | -5分 | -1星
第2次   | -10分| -1星
第3次   | -15分| -2星
第4次+  | -20分| -2星
```

### 2. 时间惩罚
- 每次错误增加30秒时间压力
- 连续错误：暂停10秒，强制阅读错误分析
- 超时惩罚：每超时1分钟扣5分

### 3. 提示限制
- 每章节最多3次提示机会
- 使用提示扣5分
- 第3次提示后不再提供帮助

### 4. 专家负面反馈
- 错误1次：警告级反馈
- 错误2次：批评级反馈  
- 错误3次+：严重警告，影响最终评级

## 三、各章节难度提升方案

### Prologue（序章）- 需求分析

#### 当前状态
- 3个线索 → 3个目标
- 简单1对1匹配

#### 提升方案
**阶段1：线索收集（增加到5个线索）**
```
1. 栏杆高度异常（原有）
2. 枯水期经常停电（原有）
3. 村民夜间看急诊（原有）
4. 【新增】桥面湿滑有青苔 → 考查"环境因素分析"
5. 【新增】老人视力不佳，看不清路 → 考查"用户特征分析"
```

**阶段2：需求分类（新增环节）**
将5个线索分类到：
- 功能性需求（照明、防护）
- 非功能性需求（可靠性、安全性）
- 约束条件（成本、环境）

**阶段3：优先级排序（新增环节）**
对需求进行优先级排序，考查：
- MoSCoW方法（Must/Should/Could/Won't）
- 风险评估
- 成本效益分析

**知识点考查**
1. 需求分类方法
2. 用户画像分析
3. 约束条件识别
4. 需求优先级排序
5. 风险评估基础

### Chapter1（第一章）- 黑箱法与筛选法

#### 当前状态
- 黑箱法：3个输入 → 1个输出
- 筛选法：简单选择题

#### 提升方案

**黑箱法升级**
```
输入端（增加到6个）：
1. 太阳能板（原有）
2. 光敏传感器（原有）
3. 扬声器（原有）
4. 【新增】蓄电池 → 考查"能量存储"
5. 【新增】充电控制器 → 考查"电源管理"
6. 【新增】LED驱动电路 → 考查"负载控制"

输出端（增加到4个）：
1. 自动照明
2. 声音警报
3. 【新增】电量指示
4. 【新增】故障报警
```

**筛选法升级 - 多维度评估**
```
评估维度增加到5个：
1. 成本（原有）
2. 可靠性（原有）
3. 【新增】维护难度
4. 【新增】环保性能
5. 【新增】用户接受度

每个维度都需要权重分配，总和=100%
```

**新增：系统分析环节**
- 绘制系统框图
- 识别关键路径
- 分析失效模式（FMEA基础）

**知识点考查**
1. 黑箱建模方法
2. 系统输入输出分析
3. 多目标决策
4. 权重分配方法
5. 筛选矩阵应用
6. FMEA基础概念

## 五、失败与重试机制

### 章节失败条件
```
满足以下任一条件视为失败：
1. 错误次数 ≥ 5次
2. 得分 < 60分
3. 超时（每章节15分钟）
4. 质量评级 < 2星
```

### 失败后果
1. **强制学习**：必须阅读知识点讲解（3分钟）
2. **降级重试**：从该章节重新开始
3. **分数惩罚**：重试扣除20%基础分
4. **成就锁定**：该章节成就无法获得

### 重试限制
- 每章节最多重试2次
- 第3次失败：建议退出，学习基础知识后再来

## 六、知识点提示系统升级

### 提示层级
```
Level 1（5分成本）：
- 提示思考方向
- 不给出具体答案

Level 2（10分成本）：
- 缩小选择范围
- 给出部分提示

Level 3（15分成本）：
- 直接给出答案
- 但扣除该题50%分数
```

### 提示内容示例
```
问题：选择合适的灯杆材料

Level 1提示：
"考虑户外环境的腐蚀性和成本因素"

Level 2提示：
"镀锌钢管和不锈钢管都能防腐，但成本差异大"

Level 3提示：
"答案是镀锌钢管，性价比最优"
```
